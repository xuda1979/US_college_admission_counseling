# Privacy & Compliance Guidelines

- **Data Minimisation:** Collect only application-related data required for evaluations. Avoid storing sensitive identity information unless necessary for counseling outcomes.
- **Access Controls:** Enforce role-based access for applicants, counselors, and administrators. Log authentication events and sensitive data reads.
- **Encryption:** Store secrets and database credentials using environment variables. Encrypt databases at rest and enforce HTTPS in all deployment environments.
- **Retention & Deletion:** Provide applicants with clear retention timelines. Support data export and deletion requests in line with FERPA-like expectations.
- **Consent:** Obtain explicit consent before using AI-generated feedback. Communicate how GPT-5 outputs are used and allow opting out of data sharing.
- **Audit Trails:** Maintain audit logs for evaluations, essay critiques, and recommendation acknowledgements to support coaching oversight.
